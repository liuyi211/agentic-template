## 3. 技术约束与规范

### 3.1 代码规范

```
- 所有代码必须通过 TypeScript 严格模式检查
- API 必须包含 OpenAPI/Swagger 注释
- 每个模块必须包含单元测试（覆盖率 >= 80%）
- 提交信息遵循 Conventional Commits 规范
```

### 3.2 数据库规范

```
- 所有表必须包含 created_at 和 updated_at
- 外键必须建立索引
- 枚举类型使用数据库约束
- 敏感字段必须加密存储
```

### 3.3 API 规范

```
- RESTful 设计，使用标准 HTTP 方法
- 统一响应格式：{ code, data, message }
- 错误码全局统一管理
- 分页参数标准化：page, pageSize
```
